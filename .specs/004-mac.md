# macOS 兼容性评估

## 评估结论

当前架构对 macOS 的兼容性：**85%**

- **15% 需要修改**：主要是 Windows 特定的 API 调用
- **85% 可以直接复用**：UI 布局、业务逻辑、Agent 系统

## 架构优势

当前架构设计良好，易于跨平台：

✅ **UI 逻辑与平台特定代码分离**
✅ **使用标准 Qt 组件**（PySide6 是跨平台框架）
✅ **布局系统清晰**（QVBoxLayout、QHBoxLayout）
✅ **业务逻辑（Agent）与 UI 解耦**

## 必须修改的部分

### 1. Windows 特定 API（P0）

**问题代码位置：** [`packages/windows-app/ui/main_window.py:10,17,236-237`](../packages/windows-app/ui/main_window.py)

```python
import ctypes
user32 = ctypes.windll.user32

# 在 show() 方法中
hwnd = self.winId()
user32.SetForegroundWindow(hwnd)
```

**影响：** 这些是 Windows 特定的 API 调用，在 macOS 上会报错

**解决方案：** 使用 `sys.platform` 进行条件编译

```python
import sys

# 条件导入
if sys.platform == "win32":
    import ctypes
    user32 = ctypes.windll.user32

    def focus_window(hwnd):
        user32.SetForegroundWindow(hwnd)
elif sys.platform == "darwin":
    # macOS 使用不同的窗口焦点 API
    def focus_window(hwnd):
        # macOS 可能需要使用 Cocoa API 或其他方法
        pass
```

### 2. 窗口行为差异（P1）

**当前代码：** [`self.setWindowFlags(Qt.WindowStaysOnTopHint)`](../packages/windows-app/ui/main_window.py:34)

**macOS 注意事项：**
- macOS 的 "Always on Top" 行为与 Windows 略有不同
- 可能需要额外的权限才能设置窗口置顶
- 建议测试并可能需要添加 `Qt.WindowType` 标志

**建议：**
```python
# 可能需要额外的标志
self.setWindowFlags(
    Qt.WindowStaysOnTopHint |
    Qt.WindowType.Window
)
```

### 3. 系统托盘适配（P1）

**当前实现：** [`packages/windows-app/ui/system_tray.py`](../packages/windows-app/ui/system_tray.py)

**macOS 差异：**
- macOS 的系统托盘称为 "Menu Bar"（菜单栏）
- 图标显示和行为略有不同
- 可能需要调整图标尺寸和样式
- `QSystemTrayIcon` 在 macOS 上支持，但需要测试

**建议：**
- 测试托盘图标在 macOS 上的显示效果
- 调整图标尺寸（macOS 建议使用 16x16 或 32x32）
- 验证右键菜单行为

## PyInstaller 详细说明

### PyInstaller 工作原理

PyInstaller 是一个将 Python 应用程序打包成独立可执行文件的工具：

1. **分析依赖**：扫描 Python 代码，找出所有导入的模块
2. **收集文件**：将 Python 解释器、依赖库、应用程序代码打包
3. **生成引导程序**：创建一个启动器，在运行时解压和执行代码
4. **创建可执行文件**：生成单个可执行文件或应用程序包

### Windows 当前配置

**配置文件：** [`packages/windows-app/Wisadel.spec`](../packages/windows-app/Wisadel.spec)

```python

    ['main.py'],
```

```python
    datas=datas,

    hookspath=[],
    hooksconfig={},
    runtime_hooks=[],
    excludes=[],
    win_no_prefer_redirects=False,
    win_private_assemblies=False,
    cipher=block_cipher,
    noarchive=False,
)

pyz = PYZ(a.pure, a.zipped_data, cipher=block_cipher)

exe = EXE(
    pyz,
    a.scripts,
    [],
    exclude_binaries=True,
    name='Wisadel',
    debug=False,
    bootloader_ignore_signals=False,
```

```bash

# 4. 验证公证
```

### 常见问题

#### 1. 依赖缺失

**问题：** 运行时提示模块找不到

**解决：** 在 `hiddenimports` 中添加缺失的模块

```python
hiddenimports = [
    'PySide6.QtWidgets',
    'PySide6.QtCore',
    'PySide6.QtGui',
    'agent.core',
    'agent.agents.polisher_agent',
    # 添加其他缺失的模块
]
```

#### 2. 数据文件缺失

**问题：** 配置文件或资源文件找不到

**解决：** 在 `datas` 中添加数据文件

```python
datas = [
    ('config/*.yaml', 'config'),
    ('assets/*.png', 'assets'),
    # 添加其他数据文件
]
```

#### 3. PySide6 插件问题

**问题：** Qt 插件（平台插件、图像格式插件）缺失

**解决：** 使用 `collect_data_files` 收集 PySide6 数据

```python
from PyInstaller.utils.hooks import collect_data_files

datas = collect_data_files('PySide6', include_pyds=False)
```

#### 4. 应用程序无法启动

**问题：** 双击 .app 文件无反应

**解决：**
1. 检查控制台输出：`/Applications/Utilities/Console.app`
2. 尝试从终端运行：`open dist/Wisadel.app`
3. 检查代码签名：`codesign -dv dist/Wisadel.app`

```python
import sys

class PlatformHelper:
    @staticmethod
    def focus_window(window):
        if sys.platform == "win32":
            import ctypes
            user32 = ctypes.windll.user32
            hwnd = window.winId()
            user32.SetForegroundWindow(hwnd)
        elif sys.platform == "darwin":
            # macOS 特定实现
            pass
        else:
            # Linux 或其他平台
            pass

    @staticmethod
    def get_system_tray_icon_size():
        if sys.platform == "darwin":
            return 22  # macOS 建议尺寸
        else:
            return 16  # Windows 默认尺寸
```
