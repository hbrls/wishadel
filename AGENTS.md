# MUST 核心规则

思考过程可以使用中文或英文，在对话中可以使用中文或英文输出；但写入仓库中的 Markdown 文件时必须使用中文。

优先讨论方案，直到方案确认并可以拆分成明确的任务，而非立即执行。当存在多个可选方案时，请用 1、2、3 编号列出。

如果内容已经写入 Markdown 文件，在对话中无需重复完整内容，仅需说明涉及的文件路径和修改要点。

有时我会直接修改仓库里的文件，所以请不要依赖上下文缓存，而是以仓库中的最新文件内容为准。

README.md 定义了项目约定和协作规范，每次加载 AGENTS.md 时应同时加载 README.md 以确保上下文一致。

每次新建 Session 时先确认当前开发环境是 Windows 还是 Mac，以便后续选择正确的工具和命令。
